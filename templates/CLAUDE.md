# CLAUDE.md — <项目名>

> 「宪法」:每个 session 开场必读。**本模板假设你装了 `coding-discipline` 插件**——通用编程纪律(先对齐 / TDD / 先根因 / 优先级评审 / git 流程)和上下文防毒(`context-hygiene`)由插件全局提供,这里**不重复**(重复 = 两个事实源,正是要防的)。本文件只填**你项目独有**的东西(带 ★),填完删掉 ★ 和本段。

## 我的文档结构(让 `context-hygiene` 知道哪能信·哪别读)
> 结构随你项目来、千奇百怪都行——只要对 agent 说清这几件:
- ★ **当下真源在哪**:agent 每 session 只认这里的现状(如 `CHANGELOG.md` 顶部 / `docs/STATUS.md`)。
- ★ **归档在哪、怎么标**:过时的塞哪、盖什么戳(如 `docs/archive/` + `status: 已被 XX 取代`);**agent 绝不主动读**。
- ★ **耐久的「为什么」在哪**(可选):如 `docs/adr/`。

## ★ 项目专属(每个新项目必填)
- ★ **这项目是什么**:解决谁的什么痛、为什么走这条路(是「我们的赌注」,不是竞品分析)。
- ★ **硬边界**:绝不做什么、绝不碰什么(脱离核心的一律不做)。
- ★ **技术栈 / 目录约定**。
- ★ **协作与部署**:CI 怎么跑、谁能部署、哪些高危操作(动库 / 上线 / 改 CI)只限谁。
- ★ **commit scope 词表**:本项目的组件名清单。
- ★ **怎么算成功**:每个节点的完成标准(可测 / 可验),防 agent 过度加工。
